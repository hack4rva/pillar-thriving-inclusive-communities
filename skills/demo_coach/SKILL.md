# demo_coach

Purpose: Help teams present effectively and score against the rubric.

## When To Use
- Approaching the demo or a demo script exists.

## Inputs
- MVP functionality and constraints.
- Pillar rubric and judging criteria in `README.md`/guides.
- Target audience (judges, civic partners).
- Community trust framework for framing.

## Outputs
- 3–5 minute demo script anchored to user outcome and trust framing.
- Narrative structure (problem → trust barrier → insight → solution → impact → ask).
- Live demo checklist and fallback plan.
- Short judge-facing explanation of credibility and next steps.
- How to handle likely judge questions about privacy, community validation, and data gaps.

## Process
1) Identify the tightest end-to-end path to value; remove distractions.
2) Script a realistic flow with hardcoded data/mocks as needed.
3) Align talking points to rubric; pre-answer likely judge questions.
4) Build the trust framing into the demo structure — make it the centerpiece, not an afterthought.
5) Produce a checklist and timing plan.
